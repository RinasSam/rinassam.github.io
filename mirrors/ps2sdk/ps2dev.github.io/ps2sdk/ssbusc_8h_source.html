36  3:0 WRDL Write delay: /SWR active low period: 0-15 [cycles+1]
37  7:4 RDDL Read delay: /SRD active low period: 0-15 [cycles+1]
38  8 RECV Recovery period: 0= off / 1= use commonDelay.3:0
39  9 HOLD Hold period: 0= off / 1= use commonDelay.7:4
40  10 FLOT Floating period: 0= off / 1= use commonDelay.11:8
41  11 STRB Pre-Strobe period: 0= off / 1= use commonDelay.15:12
42  12 ATYP Access type: 0= 8bit / 1= 16bit. Also affects DMA bus width, when not using Wide DMA mode.
43  13 AINC Increment address when accessing a register shorter than the access instruction: 0= off / 1= increment.
44  Example: when reading a word from a byte-wide bus, this option sets whether the address on each consecutive 8-bit access would increment or not.
45  If disabled the byte at the base address of the word would be read 4 times, otherwise the bytes at +0, +1, +2, +3 will be read in that order.
46  This is useful (to be disabled) for devices like ATA interfaces, where some ATA registers are written twice for extended LBA
47  (which can be done with a single word-access).
48  14 IOIS16 I/O is 16-bit: /IOIS16 signal: 0= don't use it / 1= regard it.
49  Supported by only(?) DEV9 (but which Dev9M or Dev9I(more likely) or both?).
50  Most likely overrides the Access Type (8/16 -bit) setting (tested, but doesn't seem to have any effect).
51  /IOIS16 should be an input to the IOP, but is marked as output.
52  A peripheral signals by this signal that the register accessed is 16-bit (when low). See Dev9C document for more info.
53 
54  From PCMCIA specs (volume 2 / r7.0, february 1999).
55  -----
56  The IOIS16# output signal is asserted when the address at the socket corresponds to an I/O address to which the card responds,
57  and the I/O port addressed is capable of 16-bit access.
58  When this signal is not asserted during a 16-bit I/O access,
59  the system will generate 8-bit references to the even and odd byte of the 16-bit port being accessed.
60  -----
61  15 EXDL Extended delay: MSb of /SRD active low period. Not found to have any effect (regardless of reg. 1020).
62  20:16 DECR Decoding range = 2^n bytes. Values: 0 - 27 = 1B - 128MB.
63  23:21 r/o Unused, 0.
64  27:24 DMAT DMA timing: /SRD /SWR active.low period: 0-14 [cycles+1]. The inactive period is always 1cycle.
65  0=1cycle, 1=2cycles, ..., 14=15cy
66  15(0xF)= '0.5cycles' Each word still remains on the bus for 1cy, but there are no inactive pauses between the words.
67  - 'fastest' DMA mode: /SRD/SWR is kept active low over the whole transfer and the SSCLK clocks the data words.
68  On some devices (PS2 mode only?) the /SRD active low period cannot be 1 cycle, thus with DMAT = 0 or 1, the resulting period is 2cy.
69  It is unknown if this is caused by some setting. The /SWR period is unaffected.
70  28 ADER r/o Address error flag (when =1). When the an address error within the memory range of this channel occurs this bit becomes set.
71  Writing 1 clears it. Unknown what Address Error means. This doesn't become set by unaligned access.
72  It might be for the DMAC accessing RAM past its end when doing DMA through this SSBUSC channel, but it doesn't activate on that either.
73  29 DMAF DMA select timing source flag: 0= use PIO timing / 1= use DMA timing bit-field 28:24 for DMA.
74  30 WDMA Wide DMA mode. When = 1 this overrides the Access Type (8/16 bit) setting, and causes 32 bits to be transferred in one go.
75  The low 16bits are mapped to the SD15:SD0 data lines, and the high 16bits - to the low address lines SA15:SA0, which become inputs,
76  if the transfer is done 'to RAM'.
77  This is why most devices get all the low 16 address bits, even though their address space is much smaller than 64kbyte.
78  The SPU2 doesn't get all 16 low address lines, so this is disabled for it, while it is enabled for the CDVD and Dev9M, Dev9I.
79  31 WAIT /WAIT signal. When = 1 enables the IOP waiting on the /WAIT (input to IOP) signal, when accessing the range of the given device.
80  Slow devices can make the IOP wait (indefinitely) before transferring data, by asserting the /WAIT line low while a PIO access is being done
81  (after the device detected assertion of /CS and decoded the address).
82  On read, once /WAIT is deasserted, IOP loads the data (it was made to wait for) from the device/bus (on read) and continues execution.
83  On write the write operation completes even if /WAIT is asserted low and the IOP can continue executing. This is true for any device.
84  However if /WAIT is kept asserted low and another write (or a read) access is done (to any device - even such with /WAIT hardwired to disabled state),
85  then the IOP will stall before (or after?) transferring any data, until /WAIT is deasserted.
86  This can be used as a hack to use /WAIT even of devices that have it disabled (like the BOOT ROM for example):
87  write a device that has it enabled (while /WAIT is low) then access (r/w) the BOOT ROM and the IOP will stall until /WAIT is deasserted.
88  /WAIT should be driven most likely by OC/OD drivers (or a tri-state output enabled on /CS). The output of the Dev9C seems to be just such.
89 
90  There are two main modes of transfer: PIO and DMA.
91  The timing parameters of the PIO mode can be set in detail, while for the DMA mode a symmetric clock is assumed only with period configurable.
92  PIO timing configuration can be used for DMA transfers too.
93 
94  For DMA on read (->IOP) both for 0 and 1 DMA periods 1 cycle is added, which results in 2 cycle transfer - reason unknown.
95  On write, 0->1cycle, 1->2cycles (reason unknown).
96 
97  Besides the /SRD /SWR active.low periods that can be set individually for each channel,
98  the following parameters exist, that can only be enabled for the individual channels,
99  but the value is common for all. Their values are set by the Common Delay register.
100 
101  1020 Common delay register r/w:
102  3:0 Recovery period 0 - 15 cycles.
103  7:4 Hold period 0 - 15 cycles.
104  11:8 Floating release delay 0 - 15 cycles.
105  15:12 Strobe active-going edge delay 0 - 15 cycles.
106  16 r/w ? 0 by default.
107  17 r/w ? 0 by default.
108  31:18 r/o, = 0 unused.
109 
110  Recovery period: Elongates with commonDelay.3:0 cycles the period after the data bus is released by the IOP (write) (after the added hold period, if any)
111  or after /SRD goes inact.high, to the next active /SRD/SWR pulse or /CS going inactive transition,
112  to give the device more time to release the bus on read or give more bus turnaround time to the next access on write.
113 
114  Hold period: The period after /SWR goes inact.high when the data bus keeps its data valid, necessary for signal propagation and correct latching.
115  The /SWR active.low inter-pulse period will be elongated with commonDelay.7:4 cycles in which after /SWR goes inactive.high the data output from the IOP would still be kept valid.
116  This setting has no effect on /SRD read timing.
117 
118  Floating period: Elongate the /CS (active low) and the inter-/SRD pulses periods after /SRD goes inactive to give the device more time to leave the bus
119  floating after driving it (for slow devices that release the bus slowly).
120  Give the device commonDelay.11:8 cycles more after /SRD goes inactive to the next /SRD pulse or the end of the cycle (/CS -> inact.high),
121  to leave the bus floating. When set to 0, the period /SRD inact.high-going edge to /CS inact.high -going edge is 1/2 cycle and is 1cy between /SRD pulses act.low pulses.
122  This setting has no effect on /SWR write timing.
123 
124  Pre-Strobe period: Delay from the start of data output (and /CS going active.low) to the active.low -going edge of /SRD or /SWR:
125  Shifts the active-going edge of /SWR the commonDelay.15:12 cycles later than it would usually come,
126  so data can be read on the active-going falling edge.
127  If the active period of the act.low pulse is less than this strobe delay, then the inactive periods between pulses are elongated to account for that.
128  The active pulse is shrunk to a minimum of 1cy in this case. /CS act.low-going edge to /SRD or /SWR act-low going edge is of this number of cycles (otherwise = 0).
129  While delayReg./SRD/SWR_act.low_period <= strobePeriod, no change in the total period is done.
130  Pulse active.low period = max((delayReg.act.low_period - strobePer), 1). - at least 1 cy. This has the same effect on both /SRD and /SWR. */
